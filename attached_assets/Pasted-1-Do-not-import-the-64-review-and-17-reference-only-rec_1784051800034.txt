2. Do not use connector_id to determine company vendor usage

This ranking rule is unsafe:

Vendor already used by this company
→ customer_supplier_connections.connector_id match

Multiple vendors can share the same connector:

Saval → cut_and_dry
What Chefs Want → cut_and_dry

A company using Saval does not mean it also uses What Chefs Want.

Either add an explicit nullable relationship:

platform_vendor_registry_id uuid
  REFERENCES platform_vendor_registry(id)
  ON DELETE SET NULL

to the appropriate company vendor/connection table, or omit the “already used by this company” ranking boost until that relationship exists.

Never infer vendor identity solely from connector_id.

3. A portal link is not punchout and does not populate a cart

These capabilities overstate what is verified:

populateCart: "punchout"

A portal URL is simply a portal URL. Unless the application passes the order into the vendor cart, it does not populate anything and is not punchout.

Use capability names such as:

{
  connectorId: "cut_and_dry",
  capabilities: {
    openPortal: true,
    exportOrderFile: "csv",
    populateCart: false,
    retrieveCatalog: false,
    submitOrder: false
  }
}

For the others:

{
  connectorId: "powernet_pnet",
  capabilities: {
    openPortal: true,
    exportOrderFile: false,
    populateCart: false
  }
}
{
  connectorId: "food_order_entry",
  capabilities: {
    openPortal: true,
    exportOrderFile: false,
    populateCart: false
  }
}

Capabilities can be upgraded when verified.
