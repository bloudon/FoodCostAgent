## Constraint migration safety

Make the constraint creation idempotent.

PostgreSQL does not support `ADD CONSTRAINT IF NOT EXISTS` in the same manner as `ADD COLUMN IF NOT EXISTS`.

Use a versioned migration or guarded `DO` blocks so a partially completed deployment cannot fail repeatedly due to existing constraint names.
